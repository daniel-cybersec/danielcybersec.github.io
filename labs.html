Lab 1: Detecting Brute Force Login Attempts

Category: Incident Detection

Tools Used: Linux Logs, SSH, Kali Linux

This lab demonstrates how brute force SSH Login attempts can be detected through log analysis and how Defensive actions can be applied to reduce risks.

Objectives

To identify multiple failed authentication Attempts and analyze indicators of a brute force attack.

Lab Setup

  • Target System: Linux machine with SSH enabled
  • Attack Simulation: Kali Linux
  • Log Source: /var/log/auth.log
Detection & Analysis
  • Repeated Failed password log entries detected

    • Figure 1:Repeated failed SSH authentication attempts observed in auth.log with time correlation.

    Analysis of the authentication logs revealed multiple failed SSH login attempts occurring within short time intervals from a single source IP address. Attempts include both invalid and valid usernames, indicating username enumeration behavior consistent with brute-force activity. Timestamp correlation confirmed automated or repeated login attempts.

  • Single source IP identified
  • High login attempt frequency within a short time window
Indicators of Compromise (IOCs)
  • Suspicious source IP address
  • Repeated authentication failures
  • Abnormal login attempt patterns
Defensive Response
  • Blocked the malicious IP address
  • Enabled SSH rate-limiting
  • Recommend Fail2Ban for automated protection
Lessons Learned

Log monitoring plays a critical role in detecting brute force attacks early, allowing defenders to prevent unauthorized access.

Mitigation & Recommendations

  • Implement account lockout policies after multiple failed login attempts.
  • Restrict SSh access using IP allowlists where possible.
  • Monitor authentication logs using a centralized logging or SIEM solution.

Lab 2: Detecting Port Scanning activity

Category: Network Monitoring

Tools Used:Network Logs, Wireshark

This lab focuses on identifying suspicious port scanning behavior by analyzing network traffic patterns.

Objectives

To detect abnormal connection attempts that indicate reconnaissance activity.

Detection & Analysis

  • Multiple connection attempts across different ports
  • Short time intervals between requests
  • Unusual scanning patterns identified

Defensive Actions

  • Logged scanning activity
  • Blocked suspicious IP addresses
  • Recommend IDS/IPS deployment
